# AGENTS: Documentation & Project Roadmap

## 🎯 Objective
Guidance for maintaining and interpreting project documentation.

## 🧠 Context & Rules

### 1. Document Hierarchy
- **`requirements/`**: Detailed specifications for features. Implementation must adhere to these rules exactly.
- **`tasks/`**: The active roadmap. Mark tasks as completed [x] only after the code is committed and verified.

### 2. Consistency
- Always check `oho_quiz_module.md` before implementing any UI or logic related to the quiz or trumpet stones.
- Ensure any new requirements are documented in this folder before implementation begins.

### 3. Updates
- If a technical decision deviates from the original requirement, update the documentation in this directory first to reflect the new truth.
- Keep task lists updated to prevent duplicate efforts between agents.
